读书笔记之无界面交互

May 5, 2018 | 设计

The best design reduces work.
The best computer is unseen.
The best interface is natural.
The best interface is no interface.

​ —— Golden Krishna, No Interface


麦咪的碎碎念

自从4月中旬开始准备去加拿大参展，加上四月初又做了《麦片》那个图片分享的项目，就很久没有更新这个博客了，也很久没有认真地坐下来读完一本书了。今天终于把这本放了两周的 NO INTERFACE – The Best Interface is No Interface 读了，没想到一口气居然全都读完了，只能说作者写得实在是太好了。

作者 Golden Krishna 相信 “无界面交互才是最自然的交互、最以人为本的交互” ，整本书的论述方式很幽默，也是作者自己从发现问题、分析问题、定义问题、到解决问题的过程，其中分享的设计案例也非常有趣，让我忍不住想到了很多自己生活中的实例，于是趁热写下这篇读后笔记，整理一下思路。


Foreword 前言

Why do phones ring?
Why are ringers just annying today?
How to digitally replicating the ways we talk in real life-ephemerality?

在前言中，Ellis Hamburger 丢出了这三个让人思考的问题，也算是整个《无界面交互》这本书的引子。在人们被屏幕包围的如今，”响铃” 和 “震动” 这两个 feature 算是已经被用烂了，无论什么app、都有事儿没事儿就推送个通知，响一下震一下的，以至于很多人最烦的就是他们的手机响了。


Welcome 介绍

Mark Weiser, Alan Cooper, Don Norman.

Screen-based Thinking

“What’s great about the iPhone is that if you want to check snow conditions on the mountain, there’s an app for that.”

真的无论在什么情境下我们需要的都是一个网站、一个app么？”There’s an app for that” 是不是已经成为了所有公司/服务商宣传自己的标配？你的服务最好的实现方法真的是通过一个app吗？

The Problem 问题

物联网从17年开始一路火到了现在，但什么才是真正服务于人，而不是要人去服务的设计？

真的所有的汽车都要一个配套的app吗？各种宣传app可以远程开锁，你再也不怕把你的钥匙锁在车里了。但真的有人想过我要用app开锁需要经历多少步么？我要先走向我的车，小心地拿出我的手机，唤醒我的手机，解锁我的手机，退出我当前打开的app，在app的海洋里找到我的汽车开锁app，点击它，等着它打开（或许还要等开屏广告），好！它终于打开了！我开始找点哪里可以开车门，找到了，滑动开车门，我终于把我的车门打开了。

忘记手机，忘记屏幕，忘记app。用你的直觉去想，最理想的开车门的方式是什么？我走向我的车，车门自己开了，而不是我通过十几步终于达成了我的目的。

我们需要存储更多的高清电影、蓝光电影，我们自己拍摄很多视频和照片，我们需要存储的东西越来越多，但这就只意味着我们需要更多的存储空间或更大的硬盘吗？不是的。追根溯源，经典的找到根源的方法，就是问自己5个Why：

为什么你需要更大的硬盘？因为我要存的素材和电影很多，每个都太大了；

为什么它们都那么大？因为我拍的都是高清的素材，想看的也都是高清的电影；

那为什么越是高清的视频就越大？因为编码问题吧，我也不太懂；

那为什么不能有一个更高效的编码技术，让同等质量的视频体积更小？是啊！为什么呢！
